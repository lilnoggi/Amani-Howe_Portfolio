üìåOverview

Game Title: (Working Title)

Genre: Genre-shifting adventure (2D platformer, FPS, dating-sim, horror, RPG)

Elevator Pitch

A surreal, narrative-driven game where the player (Sam) is sucked into a computer and must navigate a series of
increasingly twisted game worlds to rescue his friends - Erin, Mani, and Summer - and escape. But not everything is at it seems...

üéÆCore Gameplay

Gameplay Loop:

  • Play through different game genres across levels
  • Collect hidden items/complete tasks to influence the ending
  • Encounter story beats through dialogue and cutscenes
  • Reset after each level (with mounting tension)

Genres per Level:

  1. 2D Platformer - Cats, cute enemies, false safety
  2. 3D FPS - Space setting, combat and distrust
  3. Dating Sim - Dialogue-heavy, emotional/horror twist
  4. 3D Survival Horror - Tension, escape sequences
  5. Top-Down RPG - Boss battle, final choice

üß†Story & Structure

Main Characters:

  • Sam - Protagonist (player character)
  • Erin - Friend (creative, funny)
  • Mani - Friend (funny and grounded)
  • Summer - Innocent friend with a darker twist

Plot Summary

  • Friends get sucked into a mysterious game
  • Each level plays differently and warps their reality
  • Sam slowly realises the game is more than it seems
  • The group must try to escape, but multiple outcomes are possible

Endings:

  • Bad Ending (Default Route): "No One Escapes"

What This Taught Me:

Although, this project remains unfinished, the development process taught me a great deal about Unity, prototyping, and game design fundamentals. I focused primarily on building the first level, which was a 2D platformer.
Here's what I accomplished and learned:

  • Unity Fundamentals: I became comfortable navigating Unity's interface and working with core features like GameObjects, Rigidbody2D, and BoxColliders.
  • Basic Game Physics: I implemented gravity, jumping, and player movement using a custom PlayerMovementScript.cs, gaining an understanding of physics interactions and scripting.
  • Enemy AI Prototyping: I created basic AI behaviours using placeholder sprites:
    • Green cube = Player
    • Blue cubes = patrolling/chasing dogs
    • Red cubes = mice
    • Pink cube = Summer(NPC)
  • NPC Interaction System:
    • Introduced Summer as an NPC with basic dialogue.
    • Implemented UI that shows "Press E to interact" when the player is in range.
    • Triggered dialogue through input detection and proximity checks.
  • Scripting Patrol and Chase Logic: I scripted simple patrol patterns for enemies and implemented a chase mechanic using distance checks and direction logic.
  • Modular Level Design: I used basic tile-like cubes and templates to mock up the environment and test gameplay ideas.

While the project was not finished, it gabe me valuable hands-on experience with:

  • Writing and organising C# scripts
  • Testing and debugging movement mechanics
  • Using colliders and triggers for interaction
  • Designing gameplay logic through modular prototyping.
  • The basic of scripting AI behaviours and player-NPC interactions.

The early prototype served as a strong foundation and confidence builder, preparing me for more polished future projects and developing this project further.
